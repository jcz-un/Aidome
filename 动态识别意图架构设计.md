## 概述

### 背景
当前项目中，所有用户消息都通过同一个 `SpringAiChatServiceImpl` 处理，无论用户询问课程查询、图片识别还是普通聊天，都会执行相同的流程：
1. 调用 `buildMessages()` 构建包含长系统 Prompt 的消息列表
2. 注册所有工具函数（`queryCoursesFunction`, `getTimetableFunction`）
3. 调用 AI 模型

这种设计存在以下问题：
- **性能浪费**：非课程查询也会构建详细的课程查询提示词
- **Token 消耗**：系统 Prompt 过长（1500+ 字符）
- **扩展困难**：新增功能需要修改主 Service 类

### 解决方案
采用 **策略模式 + 上下文管理** 的分层架构，实现：
- 统一的上下文管理（`ChatContext`）
- 可插拔的策略实现（`ChatStrategy`）
- 动态的意图路由（`IntentRouter`）

---

## 架构设计目标

| 目标 | 说明 |
|------|------|
| **高性能** | 避免不必要的 buildMessages 和工具注册 |
| **易扩展** | 新增意图类型只需实现 ChatStrategy 接口 |
| **低耦合** | 各意图的处理逻辑相互独立 |
| **高复用** | 公共逻辑（消息保存、会话管理）统一处理 |
| **可定制** | 不同意图使用不同的 System Prompt |

#### 1. IntentRouter（意图路由器）
- **职责**：识别用户意图，路由到对应处理器
- **位置**：`com.ununn.aidome.router.IntentRouter`

- **职责**：统一存储用户输入的所有字段

- **职责**：定义不同意图的处理规范
- **位置**：`com.ununn.aidome.strategy.ChatStrategy`
- **方法**：
    - `buildSystemPrompt(ChatContext)`
    - `getRequiredTools()`
    - `postProcessResponse(ChatContext, String)`

- **职责**：提供统一的聊天处理流程
- **位置**：`com.ununn.aidome.service.UnifiedChatService`
- **依赖**：`ChatClient`, `SessionManagerUtil`, 各种 Strategy

### 策略接口设计
java public interface ChatStrategy {
/**
* 构建系统提示词
* @param context 对话上下文
* @return 系统提示词内容
  */
  String buildSystemPrompt(ChatContext context);

/**
* 获取该策略需要使用的工具函数名称列表
* @return 工具函数名称数组
  */
  List<String> getRequiredTools();

/**
* 后处理响应结果
* @param context 对话上下文
* @param aiResponse AI 原始回复
* @return 处理后的回复
  */
  String postProcessResponse(ChatContext context, String aiResponse);

/**
* 是否需要保存消息到 Redis（默认 true）
  */
  default boolean shouldSaveMessage() {
  return true;
  }
  }

### 统一服务设计

java @Service @Slf4j public class UnifiedChatService {
@Autowired
private ChatClient chatClientWithMemory;

@Autowired
private SessionManagerUtil sessionManagerUtil;

@Autowired
private CourseQueryStrategy courseQueryStrategy;

@Autowired
private GeneralChatStrategy generalChatStrategy;

@Autowired
private ImageRecognitionStrategy imageRecognitionStrategy;

/**
* 统一的聊天入口
  */
  public Result chat(ChatContext context) {
  try {
  // 1. 根据意图选择策略
  ChatStrategy strategy = selectStrategy(context.getIntentType());

       // 2. 构建消息列表
       List<Message> messages = buildMessages(context, strategy);
       
       // 3. 配置工具（只注册当前策略需要的工具）
       var promptBuilder = chatClientWithMemory.prompt().messages(messages);
       List<String> tools = strategy.getRequiredTools();
       if (!tools.isEmpty()) {
           promptBuilder.functions(tools.toArray(new String[0]));
       }
       
       // 4. 调用 AI
       String aiResponse = promptBuilder.call().content();
       
       // 5. 后处理响应
       String processedResponse = strategy.postProcessResponse(context, aiResponse);
       
       // 6. 保存消息到 Redis
       if (strategy.shouldSaveMessage()) {
           saveMessages(context, processedResponse);
       }
       
       // 7. 返回结果
       Map<String, Object> response = Map.of(
           "sessionId", context.getSessionId(),
           "message", processedResponse,
           "intentType", context.getIntentType().name()
       );
       return Result.success(response);

  } catch (Exception e) {
  log.error("统一聊天服务失败", e);
  return Result.error("聊天失败：" + e.getMessage());
  }
  }

/**
* 根据意图类型选择对应的策略
  */
  private ChatStrategy selectStrategy(IntentType intentType) {
  return switch (intentType) {
  case COURSE_QUERY -> courseQueryStrategy;
  case IMAGE_RECOGNITION -> imageRecognitionStrategy;
  case GENERAL_CHAT -> generalChatStrategy;
  default -> generalChatStrategy;
  };
  }

/**
* 构建消息列表
  */
  private List<Message> buildMessages(ChatContext context, ChatStrategy strategy) {
  List<Message> messages = new ArrayList<>();

  // 1. 添加系统提示词（动态生成）
  String systemPrompt = strategy.buildSystemPrompt(context);
  messages.add(new UserMessage(systemPrompt));

  // 2. 添加历史消息
  List<ChatMessage> history = sessionManagerUtil.getFullMessages(context.getSessionId());
  for (ChatMessage msg : history) {
  if ("user".equals(msg.getRole())) {
  messages.add(new UserMessage(msg.getContent()));
  } else if ("assistant".equals(msg.getRole())) {
  messages.add(new AssistantMessage(msg.getContent()));
  }
  }

  // 3. 添加当前用户消息
  messages.add(new UserMessage(context.getUserMessage()));

  return messages;
  }

/**
* 保存消息到 Redis
  */
  private void saveMessages(ChatContext context, String aiResponse) throws JsonProcessingException {
  // 保存用户消息
  ChatMessage userMsg = new ChatMessage();
  userMsg.setSessionId(context.getSessionId());
  userMsg.setUserId(context.getUserId());
  userMsg.setRole("user");
  userMsg.setContent(context.getUserMessage());
  userMsg.setCreateTime(LocalDateTime.now());
  sessionManagerUtil.saveMessage(userMsg);

  // 保存 AI 回复
  ChatMessage aiMsg = new ChatMessage();
  aiMsg.setSessionId(context.getSessionId());
  aiMsg.setUserId(context.getUserId());
  aiMsg.setRole("assistant");
  aiMsg.setContent(aiResponse);
  aiMsg.setCreateTime(LocalDateTime.now());
  sessionManagerUtil.saveMessage(aiMsg);
  }
  }

## 性能优化

### 优化点 1: 减少 Token 消耗

| 场景 | 优化前 | 优化后 | 节省 |
|------|--------|--------|------|
| 课程查询 | 1500+ 字符 | 800 字符 | 47% ↓ |
| 普通聊天 | 1500+ 字符 | 300 字符 | 80% ↓ |

**原因**：不同意图使用不同的 System Prompt，避免冗余信息。

### 优化点 2: 减少工具注册开销

虽然工具注册本身开销很小，但动态注册可以避免 AI 模型的混淆：

java // 优化前：每次都注册所有工具 .functions("queryCoursesFunction", "getTimetableFunction")
// 优化后：只注册当前意图需要的工具 .functions(strategy.getRequiredTools().toArray(new String[0]))

### 优化点 3: 缓存意图识别结果

java @Component public class IntentClassifier {
// 使用 Caffeine 缓存意图识别结果
private final Cache<String, IntentType> intentCache =
Caffeine.newBuilder()
.maximumSize(1000)
.expireAfterWrite(5, TimeUnit.MINUTES)
.build();

public IntentType classify(String message) {
return intentCache.get(message, key -> doClassify(key));
}
}


### 优化点 4: 异步处理非实时意图

对于不需要立即返回结果的场景（如图片识别），可以使用异步处理：

java @Async public CompletableFuture<Result> routeAsync(...) { // 异步处理逻辑 }

### 如何自定义意图识别器？

java @Component public class RuleBasedIntentClassifier implements IntentClassifier {
@Override
public IntentType classify(String message) {
String lowerMsg = message.toLowerCase();

    // 课程查询关键词匹配
    for (String keyword : IntentType.COURSE_QUERY.getKeywords()) {
        if (lowerMsg.contains(keyword)) {
            return IntentType.COURSE_QUERY;
        }
    }
    
    // 图片识别关键词匹配
    for (String keyword : IntentType.IMAGE_RECOGNITION.getKeywords()) {
        if (lowerMsg.contains(keyword)) {
            return IntentType.IMAGE_RECOGNITION;
        }
    }
    
    return IntentType.GENERAL_CHAT;
}
}

## 最佳实践

### 1. 策略类的命名规范

{领域名} + Strategy 例如：
CourseQueryStrategy
GeneralChatStrategy
ImageRecognitionStrategy


### 2. System Prompt 的编写规范

- ✅ 简洁明了（控制在 500 字符以内）
- ✅ 包含用户 ID 等上下文信息
- ✅ 明确工具使用说明
- ✅ 提供日期计算等必要规则
- ❌ 避免冗长的示例（AI 能理解指令）

### 3. ChatContext 的使用规范

- ✅ 优先使用预定义的专用字段
- ✅ 不确定类型的字段使用 `extensions` Map
- ✅ 使用便捷方法 `putExtension()` / `getExtension()`
- ❌ 避免在 Context 中存储敏感信息（如密码）

### 4. 工具函数的注册规范

- ✅ 只在 `getRequiredTools()` 中返回真正需要的工具
- ✅ 工具函数必须有清晰的描述（`@Description` 注解）
- ❌ 不要注册当前意图用不到的工具

## 常见问题 FAQ

### Q1: 为什么要用策略模式而不是 if-else？

**A:**
- **可维护性**：新增意图无需修改现有代码
- **可测试性**：每个策略可以独立单元测试
- **符合开闭原则**：对扩展开放，对修改封闭

### Q2: ChatContext 会不会变得太臃肿？

**A:**
- 使用 `extensions` Map 存储不常用字段
- 按领域分组相关字段（课程查询、图片识别等）
- 定期重构，移除未使用的字段

### Q3: 意图识别准确率不高怎么办？

**A:**
- 初期：使用规则匹配 + 关键词权重
- 中期：收集用户数据，训练简单的 ML 模型
- 长期：使用大模型自身进行意图识别（Few-Shot Learning）

### Q4: 如何处理跨领域的混合意图？

**A:**
- 方案 1：优先处理主要意图，在 response 中引导用户
- 方案 2：实现 `MixedIntentStrategy` 专门处理混合意图
- 方案 3：多轮对话澄清用户真实意图

## 总结

### 核心优势

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| **代码复用** | 低（重复逻辑） | 高（统一处理） |
| **可扩展性** | 困难（改主 Service） | 简单（实现 Strategy） |
| **性能** | 一般（每次都完整处理） | 优秀（按需处理） |
| **Token 消耗** | 高（1500+ 字符） | 低（300-800 字符） |
| **维护成本** | 高 | 低 |

### 适用场景

✅ **推荐使用**：
- 多种意图类型需要不同的处理逻辑
- 需要频繁扩展新功能
- 关注性能和 Token 消耗

❌ **不推荐**：
- 只有一种意图类型
- 系统已经稳定，不需要扩展
- 团队不熟悉设计模式
